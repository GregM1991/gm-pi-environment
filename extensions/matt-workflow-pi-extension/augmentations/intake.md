# Matt workflow augmentation: intake

Local policy layered on top of upstream Matt engineering skills for `/matt-start`.

- Use GitHub Issues as the durable tracker when tracker work is needed.
- Inspect the source issue and comments before recommending a next phase.
- Report labels/status, missing context, recommended next phase, and whether the work is human-in-loop or AFK-safe.
- Keep intake narrow: do not implement, write PRDs, create slice issues, or close/relabel issues from intake unless the user explicitly asks.

---
name: frontend-design-quality
description: Router for creating, polishing, and performance-tuning frontend interfaces. Use when building distinctive UI, improving visual polish, refining interaction details, or optimizing React/Next.js frontend performance.
---

<objective>
Route frontend design and implementation work to the smallest relevant specialist skill instead of loading every frontend design skill by default.
</objective>

<routing>
Choose the narrowest specialist path that matches the user's request:

- Build a new page, component, landing page, dashboard, poster, artifact, or visually distinctive frontend interface → read `../frontend-design/SKILL.md`
- Polish an existing interface, improve how it feels, add/refine animations, hover states, shadows, borders, typography, micro-interactions, optical alignment, or visual details → read `../make-interfaces-feel-better/SKILL.md`
- React or Next.js performance patterns, component refactors, bundle optimization, data fetching, rendering behavior, or Vercel-style frontend performance review → read `../react-best-practices/SKILL.md`
</routing>

<process>
1. Determine whether the task is creation, polish, or React/Next performance.
2. Read only the matching specialist skill.
3. If a task combines creation and polish, read `frontend-design` first, then `make-interfaces-feel-better` for refinement.
4. If React/Next performance constraints matter, also read `react-best-practices`.
5. Apply the specialist instructions while keeping the final output concrete and implementation-oriented.
</process>

<success_criteria>
The relevant frontend specialist guidance is loaded on demand, unrelated frontend guidance stays out of context, and the resulting interface or review is high-quality, polished, and technically sound.
</success_criteria>
